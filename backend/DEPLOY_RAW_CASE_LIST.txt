DEPLOY: Raw case list fix (required for dev + .exe to show saved cases)

Problem
-------
Saved raw mapping cases are in the VPS SQLite DB, but the live API only had
POST /api/v1/raw-mapping/cases. Without GET list, Electron cannot reload cases
on a fresh app start.

Fix (automated on VPS)
----------------------
Use scripts/vps_restart_backend.bat on the server. It will:
  1. git clone or git pull FX_TrendMaster into:
       C:\Users\Administrator\Desktop\FXTM App\FX_TrendMaster
  2. copy backend\main.py and backend\candle_store.py into:
       C:\Users\Administrator\Desktop\FXTM App\trading_gate\app
  3. set DATABASE_PATH / RAW_MAPPING_DB_PATH
  4. start uvicorn (same as your old .bat)

Replace your old restart .bat with scripts/vps_restart_backend.bat, or run it once.

Manual fix (2 files)
--------------------
Copy from this repo into your VPS app folder:

  backend/main.py   ->  trading_gate/app/main.py
  backend/candle_store.py   ->  trading_gate/app/candle_store.py

Then restart uvicorn on port 8000.

Verify
------
  curl "https://api01.apexcoastalrentals.co.za/api/v1/raw-mapping/cases?symbol=XAUUSD&limit=5"

Expected: JSON with ok:true and a cases array (may be empty if no raw cases yet).
If you see 405 Method Not Allowed, the old main.py is still in trading_gate/app.

After deploy
------------
Rebuild Electron (npm run dist) or restart npm run dev.
Map Studio -> Case Manager -> "Saved cases (from VPS)" loads from backend only.

Hard-delete orphan ranges (master case cleanup)
-----------------------------------------------
1. Deploy backend (copy main.py + candle_store.py, restart uvicorn), OR run on VPS DB directly.
2. API path (from your PC):
     python electron/scripts/cleanup_master_orphans.py --hard-delete
3. Direct DB path (on VPS RDP):
     python scripts/hard_delete_orphans_sqlite.py --db "C:\Users\Administrator\Desktop\FXTM App\trading_gate\app\market_memory.db" --execute
Keeps MACRO #152 tree; permanently removes unlinked/orphan ranges + linked events.
